# 3. Web Search

> AI 的知识有截止日期。Web Search 让它能获取实时信息——但你要知道什么时候触发、什么时候主动要求。

---

## 核心问题：知识冻结

```
AI 的训练数据 ──────────────●─────────── 时间线
                            ↑
                      知识截止日期
                  （之后的事它不知道）
```

例：GPT-4 的知识截止是 2025 年 8 月。2025 年 8 月之后的梗、新闻、产品变动，它都需要搜索才能回答。

---

## 什么会触发 Web Search？

| 类型 | 举例 | 为什么需要搜索 |
|------|------|---------------|
| **时效性问题** | "2025 年的 6 7 梗是什么？" | 可能在知识截止之后出现 |
| **本地/位置问题** | "Mountain View 附近评分高的健身房" | 信息会变化（开关店、评分变动） |
| **冷门/小众信息** | "Marquette 山的滚奶酪比赛" | 训练数据里样本少 |
| **实时事件** | "今天的股市行情" | 明确需要最新数据 |

---

## 什么不需要搜索？（预训练知识能搞定）

| 类型 | 举例 |
|------|------|
| 常识 | "手机掉汤里怎么办？" |
| 科学解释 | "为什么猫盯着墙看？" |
| 历史事实 | "旅行者 1 号唱片上录了什么？" |

**规律：** 互联网上大量存在的通用知识 → 不搜索；时效性/地域性/冷门内容 → 触发搜索。

---

## 怎么让 AI 去搜索？

两种方式：

| 方式 | 操作 |
|------|------|
| **自动触发** | AI 判断自己不够了解，自行搜索 |
| **手动触发** | 点 UI 上的搜索按钮，或在 Prompt 里写"请搜索一下" |

**实操建议：**

```
✅ 如果你需要最新信息，主动加一句：
   "请搜索最新信息后回答"

✅ 如果你发现答案可能过时，追问：
   "这是最新信息吗？请确认时效性"
```

---

## 限制：搜索不等于可靠

| 优势 | 风险 |
|------|------|
| 获取截止日期后的新信息 | 搜索结果本身可能不可靠 |
| 覆盖冷门/本地信息 | 可能引用低质量来源 |
| 提供实时数据 | 无法保证每条信息都正确 |

> **下一节将讲如何判断搜索来源的可靠性。**

---

## 本节要记住的 3 点

1. **AI 知识有截止日期** — 新事件、本地信息、冷门话题需要搜索
2. **大部分主流 AI 都有搜索能力** — ChatGPT、Gemini、Claude 均支持
3. **需要最新信息时主动要求搜索** — 别等 AI 自己判断，直接说
