# 4. Web Search Sources

> Web Search 能找到信息，但不一定找到**可靠**信息。学会引导 AI 用好来源，是避免被误导的关键。

---

## 核心问题：AI 搜索偏爱"量大"而非"靠谱"

AI 搜索引用最多的网站（按频率）：

```
Reddit > Wikipedia > YouTube > Google > Yelp > ...
```

**问题：** 社交媒体、博客、论坛的内容量远大于权威来源。如果你不指定，AI 会默认从"量大"的来源取材。

| 不引导 | 主动引导 |
|--------|----------|
| 社交媒体 + 博客 + 论坛为主 | 官方机构 + 学术研究为主 |
| 回答可能不准确 | 回答更可靠、有依据 |

---

## 实操：怎么让 AI 用靠谱来源

```
❌ "绿色市场肽类安全吗？"
   → AI 可能引用卖肽的商家网站和 Reddit 帖子

✅ "绿色市场肽类安全吗？请基于 WHO、FDA 等官方健康机构
   和经同行评审的研究来回答。"
   → AI 更可能引用世卫组织、美国 FDA、欧洲药品管理局
```

**万能句式：**

```
"请优先使用 [权威来源类型] 来回答：
 - 官方机构（政府、WHO、FDA 等）
 - 学术论文（同行评审）
 - 行业权威报告"
```

---

## 第二个陷阱：过时网页

AI 搜索到的网页可能很旧：

> 案例：问"Henderson, Nevada 适合跑步的地方"
> → AI 引用了一个 20+ 年前的网页
> → 推荐的学校现在已经不对外开放了

**对策：**

```
✅ "请确保引用的信息是近 1-2 年内的"
✅ "如果搜索结果可能过时，请注明信息日期"
```

---

## 底层原理：AI 搜索是怎么工作的

```
你 → [用户端 AI模型] → 决定需要搜索
                ↓
        [助手 AI模型] → 执行搜索 → 扫描结果
                ↓           → 过滤无关内容
                ↓           → 下载+摘要相关网页
                ↓
        [助手返回摘要] → 用户端 AI 基于摘要生成最终回答
```

**关键细节：** 用户端 AI 只看到网页**摘要**，没有读完整网页。
→ 这就是为什么 AI 有时"引用了一个网页，但结论跟网页实际内容不符"。

**对策：** 涉及重要决策时，**点开 AI 给的引用链接自己看一眼**。

---

## AI 搜索 vs 传统搜索引擎：什么时候用哪个

| 用搜索引擎（Google/Bing） | 用 AI 搜索 |
|---------------------------|-----------|
| 想快速浏览多个来源 | 想要多来源综合分析 |
| 想找到某个特定网站 | 想要复杂问题的利弊权衡 |
| 想看原始数据/直接购买 | 想节省逐个阅读网页的时间 |

**实操建议：**

```
简单查找 → Google
综合分析 → AI 搜索
重要决策 → AI 搜索 + 自己验证引用来源
```

---

## 本节要记住的 3 点

1. **主动指定来源类型** — 不指定 = AI 默认用 Reddit/论坛，指定"官方来源" = 质量飞升
2. **网页可能过时** — 重要信息要求 AI 注明时效性
3. **AI 只看摘要不看全文** — 关键引用自己点开验证
