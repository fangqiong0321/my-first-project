# 5. Using Deep Research

> 普通搜索看几个网页，Deep Research 看几十个网页 + 自主判断 + 综合报告。这是 AI 最被低估的能力之一。

---

## 三层信息获取能力对比

| | 预训练知识 | Web Search | Deep Research |
|--|-----------|-----------|---------------|
| **来源数量** | 0（已有知识） | 几个网页 | 几十个甚至更多 |
| **时效性** | 截止日期内 | 较新 | 较新 |
| **耗时** | 几秒 | 十几秒 | 几分钟到十几分钟 |
| **触发方式** | 默认 | 自动或手动 | 通常需手动选择 |
| **适合场景** | 常识、定义、基础事实 | 单一问题、实时查询 | 复杂分析、多维度综合 |

---

## 什么时候用 Deep Research？

| 用 Web Search | 用 Deep Research |
|--------------|-----------------|
| "附近评分高的健身房" | "每天步数对长期健康的影响，基于最新研究" |
| "迪拜这周天气" | "天气如何影响迪拜旅游业？综合分析" |
| 单一事实查询 | 需要权衡利弊、综合多来源 |
| 几秒能回答 | 人工做要花几小时 |

**判断口诀：**

```
一个简单问题 → Web Search
多个维度需要综合思考 → Deep Research
```

---

## Deep Research 工作流程

```
你的问题
    ↓
AI 制定研究计划（你可以审核/修改）
    ↓
并行发起多个搜索（不是一个一个搜）
    ↓
扫描结果 → 判断相关性 → 决定是否追加搜索
    ↓              ↑
    └──── 循环几轮 ─┘
    ↓
综合所有来源 → 撰写带引用的研究报告
    ↓
呈现给你
```

> 这是 **Agentic AI（自主 AI）** 的典型案例——AI 自己决定下一步搜什么、搜多少、什么时候停。

---

## 实操示例：规划万圣节鬼屋

```
Prompt：
"帮我在前院搭建一个万圣节鬼屋。
背景：
- 地点：Palo Alto, CA
- 前院面积：约 20x30 英尺
- 目标体验：适合 8-12 岁孩子，吓人但安全
- 预算：$500 以内

请做深度研究，包括：
- 当地法规和许可要求
- 安全指南（消防、电气）
- 装饰方案和预算分配
- 噪音管理"
```

**输出：** 多章节研究报告，含法规摘要、预算饼图、安全清单、时间表。

---

## 使用建议

| 建议 | 为什么 |
|------|--------|
| **给够上下文** | 你的具体情况越清楚，研究方向越精准 |
| **审核研究计划** | 大部分工具会先展示计划，明显偏了就改 |
| **不改也行** | 如果计划看着合理，直接让它跑 |
| **等它跑完** | 可能要几分钟，耐心等，不要中途打断 |
| **结果可二次利用** | Gemini 可导出为网页/信息图；其他工具可复制为文档 |

---

## 本节要记住的 3 点

1. **Deep Research = AI 版的"研究助理"** — 并行搜索几十个来源，自主决定搜索路径
2. **适合复杂、多维度问题** — 人工做要花小时的研究，AI 几分钟完成
3. **需要主动触发** — 在 ChatGPT/Gemini/Claude 中选择"深度研究"模式

---

## Module 1 总结

```
信息获取的三条路径：

预训练知识（已有知识）
    ↓ 不够用时
Web Search（实时搜索几个网页）
    ↓ 还不够用时
Deep Research（深度研究几十个网页 + 综合报告）
```

选哪个取决于：**问题的复杂度 × 对时效性的要求 × 你愿意等多久**。
